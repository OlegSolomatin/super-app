# PLAN: Order Book Trading — полный список улучшений

## 🧠 Анализ: нужна ли длительность?

**НЕТ**, для Order Book стратегий длительность не нужна в классическом понимании.

### Почему
1. **Order Book = live трейдинг.** В отличие от свечных стратегий (где есть history/virtual mode с duration_days), Order Book стратегии работают **в реальном времени** — слушают WebSocket Binance 24/7 и торгуют пока не остановишь вручную.
2. **Imbalance Scalping** — скальпинговая стратегия с быстрыми входами/выходами (секунды-минуты). Нет смысла говорить «торгуй 7 дней» — стратегия либо приносит профит, либо нет, и ты сам её остановишь.
3. **Что вместо длительности:** добавим в **шаг «Защиты»** опцию «Автостоп через N часов» как дополнительный safety switch (не основной параметр).

**Вывод:** убираем duration, оставляем бесконечную торговлю до ручной остановки.

## 📐 Дизайн каждого улучшения

### 1.1 — Выбор режима (новая модалка)

```
┌─────────────────────────────────────┐
│  Режим торговли                     │
│                                     │
│  ┌─────────────────────────────┐    │
│  │ ○ Virtual Mode              │ ← выбрано
│  │  Реальные данные Binance,   │
│  │  виртуальный баланс $1000   │
│  │  Без риска для капитала     │
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │ ○ Real Mode  [⛔ Скоро]     │ ← disabled
│  │  Реальный баланс и сделки   │
│  │  на Binance через API ключ  │
│  └─────────────────────────────┘    │
│                                     │
│         [Далее]                     │
└─────────────────────────────────────┘
```

**Реализация:** ModalBottomSheet или отдельный «шаг 0» в визарде (сдвинуть шаги).

**Тёмная тема:** PfCard с `pc.cardC`, активный = primary border.
**Светлая тема:** PfCard с `pc.cardLight`, активный = primary border.

### 1.2 — Поиск + пагинация пары

```
┌─────────────────────────────────────┐
│ 🔍 Поиск пары...              [X]  │ ← TextField
├─────────────────────────────────────┤
│ ┌─────────────────────────────┐    │
│ │ BTCUSDT        54,321.00 ₮  │    │
│ │ BTC/USDT  Vol: 1.2B         │    │
│ ├─────────────────────────────┤    │
│ │ ETHUSDT         3,456.00 ₮  │    │
│ │ ETH/USDT  Vol: 850M         │    │
│ ├─────────────────────────────┤    │
│ │ ...                         │    │
│ └─────────────────────────────┘    │
│  ↓ scroll for more (page 2/10)     │
└─────────────────────────────────────┘
```

**Реализация:** `TextField` + `ListView.builder` с `ScrollController`.
- При загрузке: `GET /trading/pairs?page=1&page_size=50&sort=volume`
- При скролле вниз: `page++`, append к списку
- При поиске: сброс пагинации, `GET ...&search=BTC`
- Backend уже поддерживает поиск и пагинацию

**Flutter компоненты:** `SearchPairDelegate` или inline `TextField` + `ScrollController`.

### 1.3 — «?» подсказки

```
  Stop Loss  [?]              -2.0%
  ─────────────────────●───────
  ╔═══════════════════════════╗
  ║ Stop Loss — автома-      ║  ← BottomSheet при нажатии [?]
  ║ тический выход из по-    ║
  ║ зиции при падении цены   ║
  ║ ниже указанного уровня.  ║
  ║                           ║
  ║ Пример: при Stop Loss     ║
  ║ -2% и входе $100, выход  ║
  ║ произойдёт при $98.       ║
  ╚═══════════════════════════╝
```

**Реализация:** `GestureDetector` с `PhosphorIcon(PhosphorIconsFill.question, size: 14)` → `showModalBottomSheet` с текстом.

**Тексты подсказок (каждому полю):**

| Поле | Текст подсказки |
|------|----------------|
| **Баланс** | «Виртуальный баланс для симуляции торговли. Средства не настоящие — вы не рискуете реальным капиталом. Можно изменить в любой момент до запуска.» |
| **Макс. сделок** | «Одновременное количество активных позиций. При 1 — только одна открытая сделка. При 3 — до 3 параллельных, если позволяет баланс.» |
| **Stop Loss** | «Автоматический выход из позиции при падении цены ниже указанного процента. Защита от крупных потерь. -1% = выход при падении на 1%.» |
| **Trailing Stop** | «Динамический стоп-лосс, который подтягивается за растущей ценой. Если цена выросла на 1%, а trailing 0.5% — стоп сдвигается на 0.5% ниже нового максимума.» |
| **Trailing Offset** | «Отступ от максимума цены, после которого активируется trailing stop. Нужен чтобы дать цене «разогнаться» перед защитой.» |
| **Max Hold** | «Максимальное время удержания одной позиции. Если за это время не сработал TP/SL — позиция принудительно закрывается. Защита от зависших сделок.» |
| **Confirmation Ticks** | «Количество последовательных тиков стакана, которые должны подтвердить сигнал перед входом. Больше тиков = меньше ложных входов, но больше задержка.» |
| **Max Spread** | «Максимальный допустимый спред (разница между лучшей ценой покупки и продажи). Если спред выше — сигнал игнорируется. Защита от неликвидных моментов.» |
| **Cooldown** | «Пауза после выхода из сделки перед следующим входом. Предотвращает повторные убыточные входы в ту же пару сразу после стоп-лосса.» |
| **Автостоп** | «Автоматическая остановка стратегии через указанное количество часов. Полезно для тестов: запустил на 2 часа и забыл — стратегия остановится сама.» |
